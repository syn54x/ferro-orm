## Key Technical Decisions

### KTD-1: `add_note` + re-raise `value_error` (not a wrapping exception, not `forward_error`)

`add_note` (PEP 678, ≥3.11) preserves the **original exception's type and traceback** and appends a diagnostic line — the most honest form of "re-raise with context". We re-raise `value_error` (the VALUE-format failure — "the original error", carrying the traceback into the user's annotation line), not `forward_error`; for the real triggers the two are the same type and message, and `forward_error` stays visible via `__context__` (implicit chaining when raising inside its `except`). No new exception type, no `raise ... from ...`.

### KTD-2: The flipped unit test is the version-independent seam

`test_annotate_func_both_fail` drives `_resolve_deferred_annotations` with a **mock** `__annotate_func__` that raises on both formats, so it exercises the fixed branch on **3.13 and 3.14 alike** — independent of the real PEP 649 machinery. The end-to-end test (Task 2) additionally pins the user-facing contract through the real deferred path (covered by the 3.14 CI lane).

### KTD-3: Nothing else in the function moves

The edit replaces exactly one line (`pass`) with a 9-line note-and-raise block. The `if "__annotate_func__" ...` guard, both `try`/`return`s, both `if "BackRef[...]"` guards, and the trailing `return namespace.get("__annotations__", {})` (the eager/non-deferred path) are untouched.

## High-Level Technical Design

```
_resolve_deferred_annotations(namespace):
    if __annotate_func__ present and no __annotations__:
        try   return annotate(1)            # VALUE     (unchanged)
        except value_error:
            if "BackRef[...]": raise         # guard     (unchanged)
            try   return annotate(2)         # FORWARDREF (unchanged)
            except forward_error:
                if "BackRef[...]": raise     # guard     (unchanged)
                value_error.add_note(...)    # #155: was `pass`
                raise value_error            # #155: surface the real cause
    return namespace.get("__annotations__", {})   # eager path (unchanged)
```
